🛡️ Project 1: HTTP Server

Learn how HTTP works by exploring request/response, headers, and status codes.

📡 What is HTTP?

HTTP (HyperText Transfer Protocol) is the foundation of web communication. Every time your browser loads a page, it sends an HTTP Request to a server, and the server sends back an HTTP Response. This server lets you see that in action.

Open DevTools → Network tab (F12) and click any link below. Watch the request and response appear in real time!

🧠 What to Observe (Open DevTools → Network Tab!)

1. Request Method

Every request has a method: GET (fetch data), POST (send data), PUT (update), DELETE (remove). Clicking links sends GET requests.

2. Request Headers

Your browser automatically sends headers like User-Agent (browser info), Accept (what formats you accept), Cookie (stored cookies). Visit /headers to see them all!

3. Response Headers

The server sends back headers too: Content-Type (what kind of data), Content-Length (how big), and security headers (we'll add these in later projects).

4. Status Codes

200 = OK, 301 = Redirect, 404 = Not Found, 500 = Server Error. Try /status/200 vs /status/500 and watch the Network tab.

5. Response Body

The response body is the actual content — HTML for pages, JSON for APIs. Compare / (HTML) vs /api/data (JSON).

6. Server Console

Check your terminal — every request is logged with its method, URL, status code, and all headers. This is your server-side view!
