Attack 1: Hidden Form (Auto-Submit)

A hidden form that submits itself to the bank's /transfer endpoint. The browser automatically includes your session cookies.

Attack 2: Image Tag (if endpoint accepted GET)

If the bank foolishly accepted GET requests for transfers, an <img> tag would do it. The browser fetches the "image" URL, which is actually a transfer request, with cookies attached. (Our bank uses POST, so this one won't work — but you should know it exists.)

Use JavaScript fetch() to send a POST request to the bank. Note: this might be blocked by SOP if the bank doesn't have CORS headers, but the request still gets sent — the browser just hides the response.
